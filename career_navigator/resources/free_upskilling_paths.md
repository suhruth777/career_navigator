# Free Ways to Build Real Data Skills

Certifications matter less than proof of applied skill. Use these to
build the skill, then prove it with a project -- not just a certificate
on a resume.

## SQL
- Practice against real, messy datasets (not toy examples) -- window
  functions, CTEs, and joins across 4+ tables are what actually get
  tested in interviews.
- Build one project that requires a genuinely complex query -- e.g. a
  customer cohort/retention analysis -- and be ready to walk through
  your query logic out loud in an interview.

## Python for data work
- Focus on pandas, not general Python syntax -- most data analyst
  interviews test data manipulation, not algorithms.
- One solid end-to-end project (clean data -> analyze -> visualize ->
  write up findings) demonstrates more than five half-finished tutorials.

## Statistics
- You need working knowledge of: mean/median/distribution, correlation
  vs. causation, basic hypothesis testing (t-tests), and how to explain
  a p-value in plain language to a non-technical stakeholder. Depth
  beyond this is rarely tested at the entry level.

## Dashboarding (Power BI / Tableau)
- Pick one tool and go deep rather than learning both shallowly. Power
  BI is more common in traditional industries (logistics, distribution,
  retail); Tableau skews more common in tech and consulting.

## Turning free learning into resume proof
A certificate alone is weak signal -- everyone can get one. What
actually moves a resume forward: a portfolio project with a real
(or realistic) dataset, a clear problem statement, and a written
walkthrough of your decisions, hosted somewhere a recruiter can
actually open it (GitHub with a README, not just a Jupyter notebook
dump).
